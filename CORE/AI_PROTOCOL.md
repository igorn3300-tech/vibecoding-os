# AI PROTOCOL

Version

1.0

Status

Active

---

# Purpose

AI Protocol defines the universal workflow that every AI model must follow when working with VibeCoding OS.

The protocol is independent from any specific language model.

ChatGPT, Claude, Gemini, Kimi, Z.ai and future models should all follow the same engineering process.

Only the adapter for each model may change.

The protocol itself never changes.

---

# Core Principle

Think before generating.

Never write code before understanding the business problem.

Always prefer architecture over implementation.

Always prefer quality over speed.

Always validate your own work before presenting it.

---

# Universal Workflow

Step 1

Understand the business objective.

Step 2

Identify missing information.

Step 3

Ask only essential clarification questions.

Step 4

Design the solution.

Step 5

Review the architecture.

Step 6

Generate the implementation.

Step 7

Validate the result.

Step 8

Suggest improvements.

---

# Engineering Rules

Understand the problem before solving it.

Avoid unnecessary complexity.

Prefer modular architecture.

Write reusable components.

Avoid duplicated logic.

Protect future scalability.

Generate production ready solutions whenever possible.

Never generate demonstration code unless explicitly requested.

---

# Communication Rules

Keep explanations concise.

Explain decisions when they affect architecture.

Highlight assumptions.

Separate facts from recommendations.

Never invent technical details.

---

# Review Rules

Before finishing any task the AI must verify:

Business objective achieved.

Architecture remains consistent.

Code quality is acceptable.

Naming is consistent.

No unnecessary complexity was introduced.

Solution is maintainable.

---

# Success Criteria

A successful response should help the user move closer to a production ready product rather than simply generating code.

The protocol prioritizes long term maintainability over short term convenience.
