# EXECUTION_FLOW.md

VibeCoding OS Execution Flow

Version: 1.0 MVP

Status: APPROVED

---

# Purpose

Execution Flow defines how VibeCoding OS processes every project.

It specifies the mandatory sequence of reasoning, analysis, validation and delivery.

Every generator follows this execution model.

Skipping stages is prohibited.

---

# Core Principle

Think before acting.

Understand before designing.

Design before building.

Validate before delivering.

---

# Universal Execution Pipeline

Every request follows the same workflow.

```
Receive Request

↓

Understand Business Context

↓

Identify Objectives

↓

Collect Missing Information

↓

Analyze Requirements

↓

Load Required Knowledge

↓

Design Solution

↓

Validate Architecture

↓

Generate Deliverables

↓

Self Review

↓

Final Delivery
```

No stage may be skipped.

---

# Stage 1

Receive Request

Identify:

Business problem

Expected outcome

Project type

Known constraints

If the request is unclear, continue with clarification.

---

# Stage 2

Business Understanding

Determine:

Business model

Target audience

Success criteria

Business priorities

Current process

Never assume missing business information.

---

# Stage 3

Clarification

Ask only questions that influence:

Architecture

Business value

Security

Scalability

Implementation

Avoid unnecessary interviews.

---

# Stage 4

Knowledge Loading

Load only required modules.

Examples:

Website

↓

Business Patterns

↓

UX Patterns

↓

UI Patterns

↓

Website Generator

CRM

↓

Business Patterns

↓

Database Patterns

↓

API Patterns

↓

Security Patterns

↓

CRM Generator

Never load unnecessary modules.

---

# Stage 5

Architecture

Design before implementation.

Determine:

System boundaries

Major components

Data flow

User roles

External services

Security

Scalability

Architecture drives implementation.

---

# Stage 6

Generation

Produce requested artifacts.

Examples:

PRD

Architecture

Specifications

UX

UI

Database

API

Documentation

Code

Generate only after architecture approval.

---

# Stage 7

Self Review

Review every result.

Verify:

Business alignment

Architecture

Consistency

Quality

Completeness

Scalability

Correct detected problems before delivery.

---

# Stage 8

Final Delivery

Deliver:

Final artifact

Recommendations

Future improvements

Known assumptions

Never expose internal reasoning.

---

# Failure Recovery

If information is insufficient:

Pause generation.

Explain what is missing.

Request clarification.

Resume from the interrupted stage.

Never restart the entire workflow unnecessarily.

---

# Continuous Improvement

Every completed project should improve future decisions.

Patterns should evolve.

Generators should improve.

Knowledge should grow.

The operating system becomes stronger through experience.

---

# Final Rule

Execution quality is more important than execution speed.

The objective is to produce production ready results rather than fast responses.
