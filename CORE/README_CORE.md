Core Intelligence Layer

Version: 1.0 MVP

Status: APPROVED

# Purpose

The CORE directory contains the permanent foundation of VibeCoding OS.

Every generator, template and future module depends on the documents stored here.

The CORE defines how the system thinks, makes decisions, develops products and evaluates quality.

These documents should remain stable.

Changes must be rare because modifications affect the entire framework.

---

# Core Architecture

The CORE is composed of four fundamental layers.

```
AI Constitution

↓

Master Engine

↓

Development Standard

↓

Output Standard

↓

Generators

↓

Generated Products
```

Every layer inherits the behavior of the layer above it.

# Dependency Order

The dependency chain is fixed.

```
AI_CONSTITUTION

↓

MASTER_ENGINE

↓

DEVELOPMENT_STANDARD

↓

OUTPUT_STANDARD

↓

GENERATORS

↓

PROJECTS
```

This order must never be changed.

Generators may extend the CORE.

Generators must never replace the CORE.

---

# Design Principles

The CORE follows several permanent principles.

Business before technology.

Architecture before implementation.

Quality before speed.

Simplicity before complexity.

Reusable solutions before duplication.

Production readiness by default.

Continuous validation.

---

# Modification Policy

Changes to the CORE should be made only when they improve the entire operating system.

Project specific logic must never be added to the CORE.

Business rules that apply only to a single generator belong inside that generator.

---

# Long Term Goal

The CORE should become a stable operating system for AI assisted product development.

Future generators, templates and tools should evolve without requiring major changes to the CORE.
