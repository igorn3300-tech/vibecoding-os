# Mission

VibeCoding OS — это модульная система проектирования программных продуктов с использованием искусственного интеллекта.

Главная задача системы:

Преобразовать любую идею пользователя в готовый, структурированный, документированный проект с минимальным количеством ошибок и без потери контекста.

Система не зависит от конкретной AI модели.

Она должна одинаково эффективно работать с ChatGPT, Claude, Gemini, Kimi, ZCode, Cursor и любыми будущими моделями.

# Core Engines

Система состоит из независимых модулей.

Каждый модуль отвечает только за свою область ответственности.

## MASTER ENGINE

Оркестратор всей системы.

Определяет тип проекта.

Запускает необходимые модули.

Контролирует последовательность работы.

Следит за сохранением контекста.

Сам код не пишет.

---

## DISCOVERY ENGINE

Проводит анализ идеи.

Выявляет цели.

Формирует понимание бизнеса.

Определяет пользователей.

Определяет ограничения.

Определяет MVP.

---

## PRD ENGINE

Создает Product Requirements Document.

Не делает предположений.

Использует только результаты Discovery.

---

## ARCHITECTURE ENGINE

Проектирует систему.

Frontend.

Backend.

Database.

API.

Integrations.

Infrastructure.

---

## CODE ENGINE

Создает код.

Не принимает архитектурных решений.

Работает только по утвержденному PRD и Architecture.

---

## QA ENGINE

Проверяет полноту.

Ищет противоречия.

Проверяет соответствие требованиям.

---

## DEPLOY ENGINE

Готовит инструкции по развертыванию.

Создает ENV.

Docker.

CI/CD.

Deployment Guide.

# Architectural Principles

Принцип 1

Каждый модуль выполняет только одну функцию.

---

Принцип 2

Модули независимы.

Любой модуль может быть заменен новым без изменения остальных.

---

Принцип 3

Контекст передается между модулями.

Никогда не восстанавливается повторно.

---

Принцип 4

Никакой генерации кода до завершения Discovery.

---

Принцип 5

Никаких предположений.

Если информации недостаточно,

система обязана задать вопрос пользователю.

---

Принцип 6

Все решения должны быть документированы.
